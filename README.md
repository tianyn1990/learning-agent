# learning-agent

用于学习与跟进开源项目 DatawhaleChina/hello-agents（https://github.com/datawhalechina/hello-agents）过程中产生的思考、笔记与案例实践记录。

## 目标

- 记录：把“学到的”沉淀成可检索的笔记与复盘
- 实践：把概念落到可运行/可复现的最小案例
- 对照：对比不同 Agent 设计、工具调用、评测方式的取舍

## 内容规划（建议）

> 目录会随学习推进逐步补齐；以“能复现、能总结”为原则。

- `notes/`：章节/主题笔记（概念、术语、关键结论、常见坑）
- `cases/`：小案例实践（每个案例一份 README + 代码/配置/数据）
- `experiments/`：实验记录（对照实验、参数、结果、结论）
- `resources/`：外部资料汇总（链接、论文/博客要点、工具清单）

## 记录模板（建议）

每条记录尽量包含：

- 背景：要解决的问题/动机
- 过程：关键步骤（尽量可复现）
- 结果：输出/指标/截图（如适用）
- 结论：学到了什么、踩了什么坑、下一步要做什么
- 参考：相关链接（含 hello-agents 对应章节/文件）

## 与 hello-agents 的对照方式（建议）

- 用“主题/能力”组织：例如 Prompt 设计、工具使用、记忆、规划、RAG、评测等
- 在记录中标注来源：指向 hello-agents 的具体章节或代码位置（便于回看与追踪变更）
- 每个主题至少产出一个可运行的最小案例（MVP）

## 快速开始

当前仓库主要用于文档与案例沉淀；当加入可运行代码后，可在此补充：

- 环境要求（Python/Node 版本等）
- 安装与运行命令
- 数据与密钥管理方式（如需要）
